# Java程序员常用的10大构建工具
[Apache Maven](http://maven.apache.org/)  ――主要用于构建Java项目的自动化工具。

[Hudson](http://hudson-ci.org/) ――用Java编写的持续集成（CI）工具。

[Jenkins](https://jenkins-ci.org/) ――用Java编写的一个开源持续集成工具。项目是在和Oracle发生争执后的来自于Hudson 的分支。

[Gradle](http://gradle.org/) ――一个开源的自动化构建系统，建立在Apache Ant和Maven Apache概念的基础上，并引入了基于Groovy的特定领域语言（DSL），而不是使用Apache Maven宣布的项目配置XML形式。

[Apache Ant](http://ant.apache.org/) ――用于自动化软件构建过程的软件工具，源于2000年初的的Apache Tomcat项目。

[SBT](http://www.scala-sbt.org/) ――用于Scala和Java项目的开源构建工具，类似于Java的Maven和Ant。

[Atlassian Bamboo](https://www.atlassian.com/software/bamboo/) ――持续集成和交付工具，它将自动化构建、测试和发布捆绑到单个流程中。

[TeamCity](https://www.jetbrains.com/teamcity/) ――来自于JetBrains的一个基于Java构建的管理和持续集成服务器。

[Grape](http://docs.groovy-lang.org/latest/html/documentation/grape.html) ――嵌入到Groovy的JAR依赖项管理器。

[Ivy](http://ant.apache.org/ivy/) ――Apache Ant项目的一个子项目，一个可传递的依赖项管理器。

[Leiningen](http://leiningen.org/) ――一个自动化构建和依赖性管理工具，用于使用Clojure编程语言写的软件项目。
